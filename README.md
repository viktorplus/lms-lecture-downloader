# LMS Lecture Downloader

Скачивает **все** лекции с Moodle LMS (`itcareerhub.de`) и сохраняет их по
предметам в том же виде, как они представлены на странице курса:

- ресурсы (`mod/resource`) → файлы (PDF/PPTX и т.п.) с оригинальными именами;
- домашние задания (`mod/assign`) и страницы-лекции (`mod/page`) → сохранённая
  HTML-страница активности;
- папки (`mod/folder`) → все файлы внутри.

---

## Требования

- [Node.js](https://nodejs.org/) 18+ (проверено на 24)
- Windows 10/11

```
node --version
```

---

## Установка (один раз)

```
npm install
npx playwright install chromium
```

---

## Настройка

1. Скопируй `.env.example` → `.env`.
2. Заполни значения:

| Переменная     | Описание                                     | Пример                          |
|----------------|----------------------------------------------|---------------------------------|
| `LMS_EMAIL`    | Логин LMS                                     | `you@example.com`               |
| `LMS_PASSWORD` | Пароль LMS                                    | `secret`                        |
| `TARGET_ROOT`  | Папка, куда сохраняются лекции                | `E:/git/ich_pdf_lesson`         |
| `LMS_BASE`     | Адрес LMS (менять только при смене домена)    | `https://lms.itcareerhub.de`    |
| `HEADLESS`     | `true` — браузер в фоне, `false` — видимый    | `true`                          |

> `.env` в git **не коммитится** (см. `.gitignore`).

---

## Запуск

```
node download_lectures.js
```

Скрипт работает полностью автоматически: проходит по всем курсам из раздела
**«Мои курсы»**, создаёт папки и скачивает недостающие лекции.

### Авторизация и токен

- При первом входе выполняется автологин по `LMS_EMAIL`/`LMS_PASSWORD`.
- Токен сессии (`MoodleSession`) сохраняется в `session.json` и
  **переиспользуется** при следующих запусках — повторный вход не требуется.
- Если войти не удаётся **5 раз подряд**, скрипт останавливается и просит
  войти вручную (запусти с `HEADLESS=false`, авторизуйся в окне браузера и
  перезапусти — счётчик сбросится автоматически после успешного входа).

### Повторные запуски (идемпотентность)

Запуск безопасно повторять. Скрипт пропускает секции, в которых уже есть
файлы, и докачивает только новое (новые курсы, новые секции).

## Структура сохранённых файлов

```
TARGET_ROOT/
├── Databases_Morning_2025_PD/
│   ├── 01 - Database 1.1_ Введение в базы данных/
│   │   ├── DA_Database_1.1.pdf
│   │   └── DA_Databases_LfS1.1.pdf
│   ├── 02 - Database 1.2_ Оператор WHERE/
│   │   ├── DA_Database_1.2.pdf
│   │   └── Databases 2025_ Домашнее задание 1.html
│   └── ...
└── ...
```

- Каждый **курс** → папка с именем предмета.
- Каждая **секция** с контентом → подпапка `NN - Название секции`.
- Файлы сохраняются с оригинальными именами; дубликаты получают ` (1)`, ` (2)`.

---

## Логи

Каждый запуск пишет подробный лог в `logs/run_<timestamp>.log` (все действия,
скачанные файлы, пропуски и ошибки) — для разбора проблем. Папка `logs/`
в git не коммитится.

## Структура проекта

| Путь                    | Назначение                                            |
|-------------------------|-------------------------------------------------------|
| `download_lectures.js`  | Точка входа                                           |
| `src/config.js`         | Конфигурация (чтение `.env`)                          |
| `src/logger.js`         | Логирование в файл и консоль                          |
| `src/auth.js`           | Авторизация, переиспользование токена, лимит попыток  |
| `src/lms.js`            | Список курсов и парсинг активностей                   |
| `src/download.js`       | Загрузка ресурсов/HTML/папок                          |
| `src/main.js`           | Оркестрация прогона                                   |
| `src/utils.js`          | Утилиты (санитизация имён и т.п.)                     |

### Служебные файлы (в `.gitignore`)

`.env`, `session.json`, `.pw-lms-profile/`, `.login_attempts`, `logs/`,
`node_modules/`.
