# LMS Lecture Downloader

Скачивает **все** лекции с Moodle LMS (`itcareerhub.de`) и сохраняет их по
предметам в том же виде, как они представлены на странице курса:

- ресурсы (`mod/resource`) → файлы (PDF/PPTX и т.п.) с оригинальными именами;
- домашние задания (`mod/assign`) и страницы-лекции (`mod/page`) → сохранённая
  HTML-страница активности;
- папки (`mod/folder`) → все файлы внутри.

---

## Требования

- [Node.js](https://nodejs.org/) 18+ (проверено на 24)
- Windows 10/11

```
node --version
```

---

## Быстрый старт (для студента)

```
git clone <repo-url>
cd get_pdf
npm install
npx playwright install chromium
copy .env.example .env        # затем впиши СВОИ логин/пароль и путь
node download_lectures.js
```

Каждый студент использует **свой** аккаунт LMS — скачиваются только его курсы
из раздела «Мои курсы». Файлы `.env`, `session.json`, профиль браузера и логи
не попадают в git, поэтому чужие учётные данные не публикуются. На пустой папке
скрипт скачивает все курсы с нуля; при повторных запусках — только новое.

## Настройка

1. Скопируй `.env.example` → `.env`.
2. Заполни значения:

| Переменная     | Описание                                     | Пример                          |
|----------------|----------------------------------------------|---------------------------------|
| `LMS_EMAIL`    | Логин LMS                                     | `you@example.com`               |
| `LMS_PASSWORD` | Пароль LMS                                    | `secret`                        |
| `TARGET_ROOT`  | Папка, куда сохраняются лекции                | `E:/git/ich_pdf_lesson`         |
| `LMS_BASE`     | Адрес LMS (менять только при смене домена)    | `https://lms.itcareerhub.de`    |
| `HEADLESS`     | `true` — браузер в фоне, `false` — видимый    | `true`                          |

> `.env` в git **не коммитится** (см. `.gitignore`).

## Запуск

```
node download_lectures.js
```

Скрипт работает полностью автоматически: проходит по всем курсам из раздела
**«Мои курсы»**, создаёт папки и скачивает недостающие лекции.

### Авторизация и токен

- При первом входе выполняется автологин по `LMS_EMAIL`/`LMS_PASSWORD`.
- Токен сессии (`MoodleSession`) сохраняется в `session.json` и
  **переиспользуется** при следующих запусках — повторный вход не требуется.
- Если войти не удаётся **5 раз подряд**, скрипт останавливается и просит
  войти вручную (запусти с `HEADLESS=false`, авторизуйся в окне браузера и
  перезапусти — счётчик сбросится автоматически после успешного входа).

### Повторные запуски (идемпотентность)

Запуск безопасно повторять. Скрипт пропускает секции, в которых уже есть
файлы, и докачивает только новое (новые курсы, новые секции).

## Структура сохранённых файлов

```
TARGET_ROOT/
├── Databases_Morning_2025_PD/
│   ├── 01 - Database 1.1_ Введение в базы данных/
│   │   ├── DA_Database_1.1.pdf
│   │   └── DA_Databases_LfS1.1.pdf
│   ├── 02 - Database 1.2_ Оператор WHERE/
│   │   ├── DA_Database_1.2.pdf
│   │   └── Databases 2025_ Домашнее задание 1.html
│   └── ...
└── ...
```

- Каждый **курс** → папка с именем предмета.
- Каждая **секция** с контентом → подпапка `NN - Название секции`.
- Файлы сохраняются с оригинальными именами; дубликаты получают ` (1)`, ` (2)`.

## Экспорт HTML‑материалов в «новый формат» (немецкий курс)

Некоторые курсы (например `IT German`) состоят из HTML‑страниц с видео и
интерактивом (H5P) без PDF. Скрипт `export_german.js` конвертирует их в
самодостаточный, читаемый офлайн формат, сохраняя весь контекст:

```
node export_german.js                      # курс по умолчанию (IT German)
node export_german.js "Имя папки курса"     # любой другой курс
```

Для каждой активности рядом с `.html` создаётся:

- **`<активность>.md`** — Markdown: заголовок, ссылка на источник в LMS,
  текст инструкций (нем.+рус.) с сохранёнными ссылками, блок «Медиа и ссылки»
  (YouTube, H5P, статьи, изображения);
- **`media/`** — скачанные YouTube‑видео (`.mp4`, до 720p) и изображения;
- **`_<Курс>_index.md`** — оглавление курса с кратким описанием каждого задания.

> Для скачивания видео нужен **yt-dlp** (и ffmpeg для склейки):
> ```
> python -m pip install yt-dlp
> ```
> H5P‑интерактив экспортировать файлом нельзя — сохраняется ссылка.

## Починка имён файлов (кодировка)

Если ранее скачанные файлы имеют «битые» имена (например `Django_13 ÐÐ¾Ð½ÑÐ¿ÐµÐºÑ.pdf`
вместо `Django_13 Конспект.pdf`) — это старый баг декодирования заголовка
`Content-Disposition`. Сам загрузчик уже исправлен; для уже скачанных файлов
есть отдельный скрипт:

```
node fix_filenames.js --dry     # показать, что будет переименовано
node fix_filenames.js           # применить
```

Он безопасен: чинит только реально битые имена, корректные (кириллица, немецкое
`ü`) не трогает, при конфликте имён пропускает. Исправленные имена правильно
отображаются и на macOS, и на Windows.

## Логи

Каждый запуск пишет подробный лог в `logs/run_<timestamp>.log` (все действия,
скачанные файлы, пропуски и ошибки) — для разбора проблем. Папка `logs/`
в git не коммитится.
